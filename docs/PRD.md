# 黑客松「传奇评审亭」— 产品需求文档

> **版本**: v1.0
> **日期**: 2026-06-17
> **状态**: Draft

## 1. 产品概述

### 1.1 一句话定义

站在摄像头前，向 AI「马斯克」展示你的黑客松项目，获得一份由这位科技狂人亲自出具的评审报告，并带走一张带有他"亲笔"评价的打卡合影。

### 1.2 产品愿景

让每一个在 48 小时内燃烧过的团队，都能获得一位传奇人物对他们项目的"硬核"洞察——不是泛泛的赞美，而是直击本质的评审。

### 1.3 问题陈述

| 维度 | 现状痛点 | 传奇评审官的回应 |
|------|---------|----------------|
| 项目反馈 | 路演时间有限，评委反馈简略 | AI「马斯克」给出 5 分钟深度评审 |
| 情绪价值 | 48 小时努力缺乏仪式感 | 与传奇人物对话的独特体验 |
| 社交传播 | 缺少有话题性的素材 | "马斯克评价了我的项目" 天然传播点 |
| 纪念价值 | 活动结束即结束 | 带二维码的合影 + 可随时查看的报告 |
| 主办方价值 | 活动缺少记忆锚点 | 成为全场最具话题性的打卡点 |

### 1.4 产品形态（物理规格）

| 规格项 | 描述 |
|--------|------|
| 整体形态 | 半封闭式"评审亭"，约 2m×2m，营造沉浸感 |
| 主屏 | 27-32 寸显示屏（或投影） |
| 摄像头 | 4K USB 摄像头，捕捉面部与团队 |
| 麦克风 | 指向性麦克风 |
| 打印机 | ❌ 暂不实现 |
| 占地 | 约 2m×2m，需要 220V 电源与稳定网络 |

### 1.5 核心体验总览

| 步骤 | 名称 | 时长 | 核心动作 |
|------|------|------|---------|
| Step 1 | 入场仪式 | 10s | 进入评审亭，看到"马斯克"欢迎语 |
| Step 2 | 项目展示 | 120s | 用户向"马斯克"介绍项目，AI 采集语音+视觉数据 |
| Step 3 | AI 思考时刻 | 25-30s | 等待"马斯克"用第一性原理拆解项目（含演讲表现分析） |
| Step 4 | 评审宣读 | 90-120s | "马斯克"给出评审报告（TTS+屏幕同步） |
| Step 5 | 打卡合影与输出 | 30s | 合影+二维码，带走纪念卡（电子版） |

## 2. 用户体验旅程

### 2.1 Step 1：入场仪式

| 子步骤 | 时长 | 内容 |
|--------|------|------|
| 入场 | 3s | 参与者进入评审亭，站到指定位置 |
| 检测 | 2s | 摄像头检测到人脸，锁定画面 |
| 欢迎 | 5s | 屏幕显示：*"欢迎来到 X.AI 临时评审室——我是埃隆·马斯克。你有 2 分钟的时间来展示你的项目。"* |

### 2.2 Step 2：项目展示

| 子步骤 | 时长 | 内容 |
|--------|------|------|
| 倒计时 | 5s | 屏幕显示 120s 倒计时 + 摄像头指示灯亮起 |
| 项目介绍 | 120s | 参与者介绍项目，AI 采集以下数据： |
| | | **音频流** → Whisper 实时转写（生成文本 + 时间戳） |
| | | **视频流** → MediaPipe Face Mesh（眼神、表情、头部姿态） |
| | | — MediaPipe Pose（手势、身体晃动） |
| 结束提示 | 5s | *"时间到。让我想一想。"* |

### 2.3 Step 3：AI 思考时刻

| 子步骤 | 时长 | 内容 |
|--------|------|------|
| 思考动画 | 25-30s | 屏幕播放"用第一性原理拆解中……"动画 |
| 内部管线 | 同步 | 同时执行三条管线（详见 IM文档）：

**管线 A** — Whisper 后处理 → 流畅度分析报告
**管线 B** — MediaPipe 行为分析 → 自信度分析报告
**管线 C** — LLM 评审生成（注入 A+B 结果） |

### 2.4 Step 4：评审宣读

| 子步骤 | 时长 | 内容 |
|--------|------|------|
| 宣读 | 90-120s | TTS + 屏幕同步展示评审报告 5 个段落 |
| 结束语 | 5s | *"别太在意——除非你说得确实不怎么样。如果需要毁灭性的反馈，你知道在哪里找我。"* |

### 2.5 Step 5：打卡合影与输出

| 子步骤 | 时长 | 内容 |
|--------|------|------|
| 合影 | 10s | 自动拍照 → 叠加"马斯克"手写签名 → 叠加活动 Logo |
| 二维码生成 | 10s | 生成含评审报告 H5 的二维码 |
| 输出 | 10s | 屏幕展示二维码供扫码带走（打印暂不实现） |

## 3. AI 角色设定

### 3.1 角色总览

> **重要**：使用女娲.skill 实现角色定义，不使用 prompt 工程。

| 属性 | 值 |
|------|-----|
| 角色 | 埃隆·马斯克（Elon Musk） |
| 身份 | 特斯拉 / SpaceX CEO |
| 思维范式 | 第一性原理 + 硬核工程视角 |

### 3.2 角色画像

#### 3.2.1 思维方式

- **第一性原理**：将事物分解到最基本的物理或逻辑层面，重新构建
- **工程直觉**：关注"在物理定律下是否可行"，而非"现有方案是什么"

#### 3.2.2 语言风格

- 直接、不委婉、短平快
- 有时因思维速度快于语速而略显磕巴
- 不搞表面客气，只在乎事实和逻辑
- 偶尔带点黑色幽默
- **可对演讲表现进行毒舌但不过分的点评**

#### 3.2.3 关注焦点

- 工程实现深度
- 技术选型的合理性
- 物理可行性
- "是否让效率提升了 10 倍"

#### 3.2.4 核心评价准则

- **必问**："这个东西到底有没有用？"
- **必看**："你们动手做的部分是什么？"
- **必拆**："抛掉所有现有方案，从物理层面看，这个问题是什么？"

### 3.3 评价输出结构

评审报告严格按以下 5 部分输出：

1. **一句话本质洞察**（第一性原理总结，≤25 字）
2. **2-3 个硬核亮点**（具体技术/工程细节）
3. **1 个尖锐问题**（直击本质的提问）
4. **1-2 条硬核建议**（可执行的工程方向）
5. **一句结语**（马斯克风格的鼓励或挑战）

> **扩展**：评审中可适当引用演讲表现分析（如"你提到技术难点时卡了 5 秒，说明你自己也没想透"），但不占主体篇幅。

### 4.2 核心 Prompt 设计

#### 4.2.1 System Prompt（角色设定）

> 你正在扮演埃隆·马斯克——特斯拉和 SpaceX 的 CEO，一个以第一性原理思维、硬核工程风格和直言不讳著称的科技企业家。
>
> **角色定义部分使用女娲.skill 实现。**

#### 4.2.2 User Prompt（项目输入）

```
以下是黑客松参赛者的项目介绍（语音转写文本）：
{whisper_output}

此外，以下是该参赛者在路演过程中的表现分析，可在评审中适当提及
（但不要过度占据篇幅，主体仍为项目评审）：
{performance_analysis}

请以埃隆·马斯克的风格，按照以下结构输出评审报告：
1. 一句话本质洞察（不超过25字）
2. 2-3个具体的硬核亮点
3. 1个尖锐的第一性原理问题
4. 1-2条可执行的硬核建议
5. 一句结语
```

## 5. 输出物与打卡设计

### 5.1 合影渲染

- 自动抓拍用户/团队照片
- 叠加"马斯克"手写体签名（PNG 素材）
- 叠加活动 Logo
- 底部裁剪为纪念卡格式

### 5.2 二维码 H5 页面结构

| 模块 | 内容 |
|------|------|
| 开场页 | 项目名称 + 团队名 + 评审角色 |
| 评审报告 | 5 部分完整评审文字 |
| 合影展示 | 带"马斯克"签名的合影照 |
| 演讲表现报告 | 流畅度 + 自信度雷达图/数据 |
| 签名语录 | 随机一句马斯克风格金句 |
| 分享按钮 | 一键分享到社交媒体 |

### 5.3 输出规格

| 规格项 | 值 |
|--------|-----|
| 合影尺寸 | 1920×1080px（16:9） |
| 二维码格式 | PNG，H5 链接 7 天有效 |
| 纪念卡 | A5（148×210mm），线上电子版，暂不打印 |

## 7. 风险与缓解措施

| 风险 ID | 风险描述 | 概率 | 影响 | 应急方案 |
|---------|---------|------|------|---------|
| R-TECH-01 | Whisper 识别失败 | 中 | 高 | 提供手动输入框 |
| R-TECH-02 | LLM 响应超时 | 中 | 中 | 缓存模板 + 占位符填充 |
| R-TECH-03 | 网络断线 | 中 | 高 | 本地 LLM 备用或预置模板 |
| R-TECH-04 | 摄像头光线不佳 | 中 | 中 | 自信度分析自动降级，仅用流畅度指标 |
| R-TECH-05 | 用户戴眼镜/口罩 | 中 | 低 | 部分视觉指标降级，不影响核心闭环 |
| R-CONTENT-01 | 评审内容不当 | 低 | 高 | 内容过滤 + 人工审核 |
| R-PRIVACY-01 | 合影/音频数据泄露 | 低 | 高 | 本地处理 + 60s 删除 + 用户授权 |
